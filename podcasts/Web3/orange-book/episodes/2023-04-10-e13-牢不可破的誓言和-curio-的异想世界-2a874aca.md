## Text

这期博客有些不一样，我想你从标题也能看出来，什么叫 13⅔ ？原因是两个月前的录音丢了前三分之一。黑天鹅真是无处不在啊。但是我听完后面三分之二，还是觉得很精彩，于是有了这一期。这期博客的嘉宾一嘉应该是最年轻的一位，他们在做一款类似“链上文明”的游戏 Treaty 。你可以在游戏中和其他玩家签订各种条约，就像现实国际世界里的条约一样，互不侵犯，军事联盟，禁运协议，关税协议。不同的是，链上的条约是自动执行的，牢不可破的誓言。如果你也是 4x 游戏爱好者，说到这里我想你应该能明白这会打开多么巨大的崭新游戏空间。嘉宾介绍：一嘉：Curio 的创始人，on-chain game 的先驱探索者。相关链接：Curio 的网址 https://www.curio.gg/Highlight：on-chain game 中的策略游戏更像 MMO ，这里有更多交易，guilds，相对自由。我们团队都是刚毕业，或者辍学，或者没毕业，平均年龄22岁左右。on-chain game一方面是非常有趣的 to C 的应用，另一方面它需要非常前沿的基础设施做支撑，因为它是 Crypto 世界里最 gas intensive 和最 compute intensive 的应用，也意味着这很可能代表着 Crypto 的未来。玩家不会在乎这个游戏是否去中心化，他们更在乎的是游戏体验，包括如果玩家想创造内容，他作为创造者的体验。目前主要有两个挑战，一个是我们低估了把智能合约做的“可组合”的难度，另一个是现有的基础设施还是性能不够，影响了玩家的游戏体验。游戏数据哪些是一定要上链的？我觉得取决于哪些数据会影响 UGL （用户生成逻辑）和可组合性。链上强制执行的条约当然和真实世界是不一样的，但是即使真实世界，在不同阶段和场景下，条约的执行也是不同的。链上也是如此。on-chain game 和传统游戏都可以玩纯粹的口头条约和 power game ，但是 on-chain game 额外增加了一个新选项，那就是能够强制执行的条约，有了新的设计可能。Carmack 做 doom 的经历对我们很有启发性，如果我们能做出性能远超其他 on-chain game 的引擎，加上 treaty 这种 social contract 带来的新玩法，也许是一条路。on-chain game 的生态里最缺的一是技术基础设施，二是为 web 3 用户之外的玩家提供 onboarding 服务的平台，有点像 steam 的角色。
